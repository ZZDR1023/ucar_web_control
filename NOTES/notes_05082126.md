## 已完成的本地代码改动

### 语音解析与动作序列

- `web_panel/voice_control.py`
  - `左转90度` 默认目标保持为 90 度，不再默认乘 1.15。
  - `build_route_point_aliases(route_name, index, include_generic=False)` 支持中文数字和当前路线泛化别名。
  - 带路线名示例：`t1中的3号巡逻点`、`t1第三个点`。
  - 当前路线泛化示例：`3号巡逻点`、`第三个点`。

### Web 后端导航逻辑

- `web_panel/server.py`
  - ROS 节点改为稳定名：`rospy.init_node("web_panel_server", anonymous=False, disable_signals=True)`，避免不断生成 `/web_panel_server_xxx` 匿名节点注册。
  - `run_shell()` 改为 `subprocess.Popen(..., start_new_session=True)`，超时时 `os.killpg()` 清理进程组。
  - `nav_nodes_running()` 不再只信 `rosnode list`，会对 `/move_base`、`/amcl`、`/map_server` 分别 `timeout 2 rosnode ping`。
  - `start_nav_processes()` 改为：
    - `rosnode cleanup` 最多等 3 秒；
    - 然后 `setsid -f bash -lc '... exec roslaunch nav_clean navigation_runtime.launch'`；
    - 避免 `roslaunch` 作为 Web shell 子进程退出时杀掉 `map_server/amcl/move_base`。
  - `/api/goal` 和语音 `goal` 在未收到 AMCL 位姿时只排队，不直接发布给 `move_base`。
  - `amcl_cb()` 收到 `/amcl_pose` 后，如果有 `pending_goal` 且当前不是手动模式，会自动发布排队目标。
  - `api_set_pose()` 已修正：设置 AMCL 位姿时不清空 `pending_goal`；如果有排队目标，返回 `queued_goal: true`，让 AMCL 位姿出现后继续自动发目标。

### Web 前端巡逻 UI

- `web_panel/templates/index.html`
  - `使用当前目标` 改名为 `使用当前地图目标`。
  - 新增 `使用小车位置`，使用 `latestStatus.pose` 作为候选巡逻点。
  - 如果没有 AMCL 位姿，会提示“还没有 AMCL 小车位置，先确认导航定位正常”。
