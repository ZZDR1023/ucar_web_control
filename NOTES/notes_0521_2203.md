## 本轮本地代码改动

### 1. 远距离转弯延迟

文件：`web_panel/person_follow_adapter.py`

新增/调整参数：

```python
turn_delay_height_ratio = 0.30
far_turn_scale = 0.18
turn_error_smoothing = 0.70
```

控制逻辑变化：

- 先对 `center_error` 做平滑，得到 `smoothed_center_error`；
- 当人体框高度比例低于 `turn_delay_height_ratio` 时，压低转向量；
- 这样人在远处刚转弯时，小车不会马上切弯，而是继续跟近，接近转弯点后再明显转向。

### 2. 启动响应优化

文件：`web_panel/server.py`

`start_person_follow_services()` 原来每轮会调用：

```text
/person_follow/start_person_detect
/person_follow/get_detect_info
/person_follow/start_person_follow
```

现在启动循环不再阻塞调用 `/person_follow/get_detect_info`，只保留：

```text
/person_follow/start_person_detect
/person_follow/start_person_follow
```

目的：避免 Web 状态已经能显示 `cx/cy`，但启动链路仍在同步等待信息服务，导致运动控制晚很多才打开。

### 3. 近距离避障减速

文件：`web_panel/person_follow_adapter.py`

新增参数：

```python
obstacle_slow_margin = 0.05
obstacle_min_linear_scale = 0.25
```

逻辑：

- 原有 `front_stop_distance=0.35` 的停车保护保留；
- 当前方距离小于 `front_stop_distance + 0.05` 且尚未小于停车阈值时，按比例降低线速度；
- 用户要求“参数别设置太大，离 4、5cm 再减速”，所以只设置 5cm 的减速带，不扩大避障范围。

## 下次上电后的验证顺序

### 1. Web 服务与跟随状态

```bash
curl -s http://10.241.30.179:8080/api/person_follow/status
```

预期未启动时：

```json
{"ok": true, "person_follow": {"running": false}}
```

点击 Web “开始跟随”后，重点看：

```text
running=true
follow_enabled=true
detected=true
command.linear_x > 0
```

### 2. 检查启动延迟是否改善

点击开始后，如果页面已经显示 `cx=xx, cy=xx`，应很快看到 `command.linear_x > 0`。

如果仍然约 10 秒才动，继续查：

```bash
systemctl status ucar_web.service --no-pager
curl -s http://10.241.30.179:8080/api/person_follow/status
```

并看 `/tmp/person_follow.log` 是否有模型加载或相机初始化延迟。

### 3. 检查提前转弯是否改善

测试方式：

1. 人在车前方较远处开始转弯；
2. 观察车是否先继续向人的旧路径靠近，而不是立刻跟着画面偏移切弯；
3. 接近人的转弯点后再观察是否开始转弯。

如果仍然提前转：

- 降低远处转向响应：减小 `far_turn_scale`；
- 或提高 `turn_delay_height_ratio`，让“远处弱转向”的区间更长。

如果转弯太晚：

- 增大 `far_turn_scale`；
- 或降低 `turn_delay_height_ratio`。

### 4. 检查避障减速

当前逻辑只在停车阈值前 5cm 减速：

```text
front_stop_distance = 0.35
obstacle_slow_margin = 0.05
```

含义：

- `< 0.35m`：停车；
- `0.35m ~ 0.40m`：渐进减速；
- `> 0.40m`：不因前方障碍减速。

如果现场觉得减速过早，不要改 `front_stop_distance`，先把 `obstacle_slow_margin` 从 `0.05` 降到 `0.04`。
