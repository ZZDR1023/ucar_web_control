# ROS 智能小车项目推荐清单

记录日期：2026-04-30

## 一、推荐总项目

### 基于 ROS 的多模态智能服务机器人系统设计与实现

该项目基于科大讯飞 ROS 小车已有功能包，综合使用底盘控制、激光雷达、摄像头、语音识别、导航、人体跟随等模块，设计并实现一个具备感知、控制、交互和自主移动能力的智能服务机器人系统。

项目最终目标：

- 实现无线远程控制小车运动。
- 实时监控机器人运行状态。
- 使用激光雷达完成室内建图。
- 基于地图实现自主定位与导航。
- 使用语音命令控制机器人。
- 使用摄像头实现人体跟随。
- 将多个功能模块融合成完整服务机器人系统。

## 三、阶段 1：无线遥控小车

### 目标

实现电脑通过无线网络远程控制实体小车运动。

### 已验证基础

- 小车可连接手机热点。
- 电脑可通过无线 SSH 登录小车。
- ROS Master 可正常启动。
- `/cmd_vel` 控制链路可用。
- 小车已经能够响应旋转命令。

### 可继续完善

- 编写安全控制脚本。
- 加入最大线速度和最大角速度限制。
- 加入自动停止保护。
- 支持前进、后退、左转、右转、停止。
- 支持键盘实时控制。

### 展示效果

电脑通过 Wi-Fi 远程控制小车移动。

## 四、阶段 2：机器人状态监控面板

### 目标

实时查看小车运行状态，提升系统可观测性。

### 可监控话题

| 话题 | 含义 |
|---|---|
| `/odom` | 里程计，包含位置和速度信息 |
| `/imu` | 惯性测量单元，包含姿态、角速度、加速度 |
| `/battery_state` | 电池状态 |
| `/cmd_vel` | 当前速度控制指令 |
| `/tf` | 坐标变换关系 |

### 方案 A：命令行监控

编写 Python ROS 节点，订阅 `/odom`、`/imu`、`/battery_state` 等话题，在终端实时打印机器人状态。

示例显示内容：

```text
线速度: 0.08 m/s
角速度: 0.60 rad/s
当前位置: x=..., y=...
电池状态: ...
```

### 方案 B：Web 状态面板

使用 Flask、FastAPI 或 rosbridge 实现网页端监控界面。

可显示内容：

- 当前连接状态
- 线速度和角速度
- 里程计位置
- 电池信息
- IMU 姿态
- 控制按钮

### 展示效果

浏览器实时显示机器人状态，并支持简单控制。

## 五、阶段 3：激光雷达建图

### 目标

使用激光雷达扫描环境，生成室内二维栅格地图。

### 相关功能包

- `ydlidar_ros_driver`
- `ucar_map`
- `cartographer_start.launch`
- `ucar_mapping.launch`

### 基本流程

1. 启动底盘驱动。
2. 启动激光雷达。
3. 启动建图算法。
4. 遥控小车在环境中缓慢移动。
5. 在 RViz 中观察地图生成过程。
6. 保存生成的地图。

### 可能使用的命令方向

```bash
roslaunch ucar_controller base_driver.launch
roslaunch ydlidar_ros_driver ...
roslaunch ucar_map ucar_mapping.launch
rosrun map_server map_saver -f my_map
```

### 展示效果

小车边移动边构建室内地图，并在 RViz 中实时显示地图。

## 六、阶段 4：自主定位与导航

### 目标

让小车根据已保存地图，自主移动到指定目标点。

### 相关功能包

- `ucar_nav`
- `ucar_navigation.launch`

### 核心功能

- 加载已保存地图。
- 使用 AMCL 完成定位。
- 使用 move_base 完成路径规划。
- 支持局部避障。
- 支持目标点导航。

### 基本流程

1. 启动底盘驱动。
2. 启动激光雷达。
3. 加载地图。
4. 启动导航系统。
5. 在 RViz 中设置初始位姿。
6. 在 RViz 中点击目标点。
7. 小车自主规划路径并移动到目标点。

### 可扩展功能

- 固定点巡航。
- 多目标点顺序导航。
- 返回起点。
- 服务点导航，例如“去门口”“去桌子旁”。

### 展示效果

在 RViz 中点击目标点后，小车自动移动到指定位置。

## 七、阶段 5：语音控制机器人

### 目标

当前先实现 Web 面板语音控制 MVP：浏览器使用 Web Speech API 做中文语音识别，把识别文本发给后端 `/api/voice_command` 解析执行。这样不需要在小车没电时安装新 ROS 语音包，也能复用 Web STOP、障碍拦截、巡逻和导航目标逻辑。

科大讯飞离线语音识别模块仍可作为后续增强方向：如果后续确认麦克风阵列和 `xf_mic_asr_offline` 已稳定运行，再把它的识别结果接入同一个后端解析器，而不是另写一套直接发 `/cmd_vel` 的控制链路。

### 相关功能包

- 当前 MVP：浏览器 `SpeechRecognition` / `webkitSpeechRecognition` + Web 后端 `/api/voice_command`
- 后续可选：`xf_mic_asr_offline`

### 可实现语音命令

| 语音指令 | 机器人行为 |
|---|---|
| 前进 | 发布 `/cmd_vel` 前进 |
| 后退 | 发布 `/cmd_vel` 后退 |
| 左转 | 发布 `/cmd_vel` 左转 |
| 右转 | 发布 `/cmd_vel` 右转 |
| 停止 | 发布零速度 |
| 开始巡逻 | 启动当前巡逻路线 |
| 暂停巡逻 | 暂停巡逻并停车 |
| 继续巡逻 | 恢复巡逻 |
| 停止巡逻 | 停止巡逻并停车 |
| 去门口 | 导航到已保存巡逻点 |
| 跟着我 | 后续人体跟随阶段再接入 |
| 结束跟随 | 后续人体跟随阶段再接入 |

### 实现方式

1. Web 前端调用浏览器语音识别，或允许手动输入命令文本。
2. 后端 `web_panel/voice_control.py` 解析关键词。
3. 手动移动命令只使用低速参数，并通过 Web 后端状态机发布。
4. `停止`、巡逻和导航目标复用现有 `/api/stop`、巡逻管理器和 `move_base` 启动/发 goal 逻辑。
5. 后续如果接入 ROS 语音识别话题，也应复用同一个解析器和执行入口。

### 简单版效果

语音或文本控制小车前进、停止、左转、右转；浏览器不支持 Web Speech API 时仍可通过输入框执行。

### 进阶版效果

语音控制机器人导航到已保存巡逻点，并可控制巡逻开始、暂停、继续、停止。

## 八、阶段 6：人体跟随

### 目标

使用摄像头识别人，并让小车自动跟随目标。

### 相关功能包

- `person_follow`
- `ucar_camera`

### 核心流程

1. 启动摄像头。
2. 启动人体检测或人体跟随节点。
3. 获取人体在图像中的位置。
4. 根据目标横向偏移控制角速度。
5. 根据目标距离控制线速度。
6. 发布 `/cmd_vel` 控制小车运动。

### 安全要求

- 控制速度要低。
- 保持安全距离。
- 丢失目标时自动停止。
- 避免误检导致小车异常运动。

### 展示效果

人在小车前方移动时，小车可以自动跟随。

## 九、阶段 7：多模态综合系统

### 目标

将无线控制、状态监控、建图、导航、语音控制、人体跟随融合成完整智能服务机器人系统。

### 推荐系统结构

```text
语音识别 / 键盘控制 / Web 控制
        ↓
    任务管理节点
        ↓
底盘控制 / 导航 / 人体跟随 / 状态监控
```

### 系统模式

| 模式 | 功能 |
|---|---|
| 手动模式 | 键盘或网页控制 |
| 建图模式 | 启动 SLAM 建图 |
| 导航模式 | 自主导航到目标点 |
| 语音模式 | 通过语音执行命令 |
| 跟随模式 | 自动人体跟随 |
| 停止模式 | 立即停止所有运动 |

### 综合演示场景

```text
用户说：“小车，去门口。”
机器人自主导航到门口。

用户说：“跟着我。”
机器人切换到人体跟随模式。

用户说：“停止。”
机器人立即停止运动。
```
