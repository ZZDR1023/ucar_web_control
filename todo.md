# ROSCAR1 项目 TODO

Web 控制面板已经基本结束。下面记录基于这台 ROS1 小车还能继续做的项目方向，适合作为课程设计、毕业设计、比赛展示或长期迭代任务。

## 1. 视觉巡线 / 道路跟随

目标：让小车按照地面路线图行驶，识别车道线、虚线、斑马线等视觉特征。

当前落地状态：

- 已新增 Web 视觉巡线第一版：`web_panel/line_follow.py`。
- 已接入 Web 面板预览、调试图、颜色/ROI/速度参数和低速控制开关。
- 默认只预览，不控车；必须点击 `启用巡线控制` 才会写入 `/cmd_vel`。
- 下一步需要铺设清晰地面线，先用调试图确认识别稳定，再做实车低速闭环测试。

适用场景：

- 地上铺了类似道路地图的图纸或地垫。
- 路线包含车道线、中心线、虚线、斑马线、转弯口。
- 小车需要沿线行驶，而不是依赖激光雷达建图导航。

关键判断：

- 这种路线不是靠激光雷达来走的，而是靠摄像头来看地面。
- 雷达主要负责安全避障，不能直接识别地上的车道线和斑马线。
- 如果当前摄像头角度看不到地面，需要调整相机朝向，或增加一个朝下摄像头。

可做内容：

- 使用 OpenCV 识别地面白线、黄线或黑色道路边界。
- 提取车道中心线，根据偏差控制 `/cmd_vel`。
- 识别虚线、斑马线、停止线等特殊区域。
- 在路口根据标志或规则左转、右转、直行。
- 结合雷达，如果前方有障碍物则停止，障碍消失后继续循迹。

产出：

- `web_panel/line_follow.py` 视觉巡线模块。
- 地面路线识别算法说明。
- 循线行驶演示视频。
- 可扩展的路口规则配置。

## 2. 自主巡逻机器人

目标：让小车按固定路线自动巡逻，并在异常时停止或上报。

可做内容：

- 建立多个巡逻点，例如门口、桌边、走廊端点。
- 按顺序自动导航到每个点。
- 到点后停留拍照或记录状态。
- 遇到障碍时等待、绕行或跳过当前点。
- 记录每轮巡逻耗时、失败点和电量变化。

产出：

- 一套 `patrol_node.py` 巡逻节点。
- 一份巡逻路线配置文件。
- 一段完整演示视频。

## 3. 语音控制移动机器人

目标：通过语音让小车执行前进、停止、去某个地点等任务。

可做内容：

- 识别简单命令，例如“前进”“停止”“去门口”“回来”。
- 把语音命令转换成 ROS 控制或导航目标。
- 增加语音播报，例如“已收到目标”“前方有障碍”。
- 支持离线语音识别，减少网络依赖。

产出：

- 语音控制 ROS 节点。
- 命令词表。
- 语音交互演示。

## 4. 视觉目标跟随

目标：小车通过摄像头识别人或指定物体，并保持一定距离跟随。

可做内容：

- 使用相机识别人脸、人体或颜色标记。
- 根据目标在画面中的位置控制转向。
- 根据目标大小估计距离，控制前进和停止。
- 结合雷达，目标前方有障碍时停止。

产出：

- `vision_follow_node.py`。
- 摄像头识别与 `/cmd_vel` 控制闭环。
- 人体或颜色目标跟随演示。

## 5. 视觉巡检与拍照记录

目标：小车到达指定位置后自动拍照，形成巡检记录。

可做内容：

- 在多个导航点自动拍照。
- 给图片加时间戳、地点名和电量信息。
- 保存巡检结果到本地目录。
- 后续可接入缺陷检测，例如识别门是否打开、物体是否存在。

产出：

- 巡检照片数据集。
- 自动拍照节点。
- 巡检报告生成脚本。

## 6. SLAM 建图与地图质量评估

目标：围绕建图效果做完整实验，而不是只使用已有地图。

可做内容：

- 对比不同建图路线对地图质量的影响。
- 记录 `/scan`、`/odom`、`/tf` 数据。
- 分析地图漂移、闭环失败、障碍边缘模糊等问题。
- 输出适合导航的封门版地图。

产出：

- 多组地图文件。
- 建图实验报告。
- 地图质量对比图。

## 7. 导航参数调优实验

目标：系统研究 DWA、costmap、AMCL 参数对导航效果的影响。

可做内容：

- 对比不同速度、膨胀半径、目标容差。
- 统计到达率、平均耗时、卡住次数、碰撞风险。
- 设计窄通道、绕障、直线到点等测试场景。
- 给出一套稳定参数。

产出：

- 参数实验表格。
- 自动测试脚本。
- 推荐导航参数配置。

## 8. 动态避障增强

目标：让小车在有人走动、临时障碍出现时更可靠地避障。

可做内容：

- 仅使用雷达实现前方动态障碍检测。
- 区分近距离急停、远距离减速、障碍清除恢复。
- 将障碍风险分成前方、左前、右前三个区域。
- 后续可接入深度相机补充低矮障碍。

产出：

- `obstacle_guard_node.py`。
- 动态避障测试记录。
- 不同障碍场景的表现对比。

## 9. 自动回充或低电量返航

目标：小车低电量时自动返回固定位置。

可做内容：

- 监听 `/battery_state`。
- 设置固定回充点或停车点。
- 电量低于阈值时拒绝新任务并返航。
- 到达回充点后停止并播报。

产出：

- 低电量任务管理节点。
- 回充点配置。
- 低电量返航演示。

## 11. ROS 机器人任务调度系统

目标：做一个独立于 Web 的任务调度层，统一管理导航、拍照、等待、语音等动作。

可做内容：

- 设计任务格式，例如 `go_to`、`wait`、`photo`、`speak`。
- 支持任务队列、取消、暂停、恢复。
- 每个任务有状态：等待中、执行中、成功、失败。
- 失败时执行重试或安全退出。

产出：

- `task_manager_node.py`。
- YAML/JSON 任务配置。
- 一套可复用任务执行框架。
