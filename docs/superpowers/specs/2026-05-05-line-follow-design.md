# 视觉巡线设计说明

## 目标

在现有 Web 控制面板中加入视觉巡线能力，让小车通过摄像头识别地面路线，并在确认识别稳定后低速道路跟随。

第一版以安全和可调试为主：默认只做识别预览，不主动控制底盘。用户必须在 Web 页面明确开启“巡线控制”后，后端才会以低速写入现有 `/cmd_vel` 发布循环。

## 范围

本次实现包含：

- 从现有摄像头缓存读取最新 JPEG 帧。
- OpenCV 识别黑线、白线、黄线三类地面路线。
- 输出线中心偏差、像素中心、检测面积、是否丢线、建议线速度和角速度。
- Web 页面显示识别叠加图、状态和参数；默认颜色为黑线，默认 ROI 为画面高度的 `0.45-0.95`，用于直接识别黑/深灰可走路面。颜色识别使用 HSV 掩码参数 `h_min/h_max/s_min/s_max/v_min/v_max`，便于现场按光照精细调节。
- 提供巡线参数接口：颜色、ROI 起止比例、线速度、角速度增益、是否启用控制。
- 保留雷达安全停止：前方障碍触发时，巡线控制立即停下并退出使能。

本次不做：

- 路口规则、斑马线语义、红绿灯识别。
- 完整路口规则、路线选择和多分支决策。当前只实现黑线路径跟踪、红/黄长实线禁压、短虚线允许跨越。
- 自动选择路线颜色。
- 云台控制。当前硬件只发现普通 USB 摄像头，没有 pan/tilt/servo 控制接口。
- 高速闭环行驶。

## 架构

新增 `web_panel/line_follow.py`，只负责纯视觉算法和参数校验，不依赖 ROS。它接收 BGR 图像后先转换到 HSV 空间，按 ROI 裁剪后用 `cv2.inRange` 根据 HSV 上下限生成 mask，再计算中心偏差并生成叠加调试图。

黑线模式当前不是简单取最大轮廓：它会从 ROI 底部向远处追踪黑/深灰可走区域，优先保持与上一帧稳定目标中心连续；左右红/黄边界能组成通道时优先使用通道中心，并根据边界弯曲方向给小幅预测转向。分叉处会过滤底部极细灰边缘，避免黄线边缘噪声把目标点拉到非可走区域；如果中心没有路但侧向存在足够宽、连续的近场分支，也允许该分支作为初始路径。单帧相反方向候选不会立即翻转角速度，除非反向证据连续出现或偏差很大。

当近场没有稳定路线、远处候选连续左右跳或红区明显收缩时，算法会把这视为路口找路状态：立即锁定当前搜索方向并固定原地同向转向。这里没有最短锁定时间；之后只要连续 3 帧看到居中的近场宽路就解除。若可走路径面积相对上一稳定帧明显下降，但上一帧已有明确分支方向，则沿上一分支方向提前原地转向，避免红色可走区域完全消失后再搜索。锁定期间黄/红禁压接触只能保持停车，不能把锁定角速度反向覆盖；临时看到的相反侧路径也不会刷新上一分支方向。

黑线 HSV 初筛不再做全局自动亮度收紧；如果初筛已经形成有边界路径，就直接按 HSV mask 追踪。只有初筛被整片低饱和灰地面淹没、无法形成路径时，才启用暗部对比 fallback，用于兼容黑线画在灰地面上的场景。

`web_panel/server.py` 负责把算法接入现有相机缓存和 ROS 控制状态。巡线控制不新建独立 ROS 节点，避免多个发布者抢 `/cmd_vel`；启用时直接写入 `CURRENT_LINEAR_X` 和 `CURRENT_ANGULAR_Z`，由现有 `cmd_loop()` 继续 20Hz 发布。

前端在 Camera 卡片下新增“视觉巡线”区域，显示叠加图、识别状态和参数控件。用户可以先只看预览，再手动开启控制。

## 安全策略

- 默认 `enabled=false`。
- 最大线速度限制为 `0.10 m/s`。
- 最大角速度参数限制为 `0.60 rad/s`；当前巡线跟随阶段内部上限为 `±0.42 rad/s`，丢线搜索角速度为 `0.24 rad/s`，分叉锁定搜索角速度随 `angular_gain` 变化，约为 `angular_gain * 0.80`，范围 `0.14-0.42 rad/s`。
- 丢线、近场没有可走路径、分叉搜索锁定、红区明显收缩或大偏差急转时线速度归零，只允许原地搜索或对准；近场连续可走区域稳定后才恢复前进。
- 红/黄禁压接触优先级最高，会清零线速度；方向连续性和弯道预测不能覆盖禁压停车。禁压接触后的角速度按底部接触侧脱离：右侧接触左转，左侧接触右转，避免继续朝黄线方向旋转。
- 如果当前没有检测到任何可走黑/灰路线，同时底部看到黄/红禁压线，则停车等待且不旋转；蓝色区域在黄线外侧，不作为可走路线。
- `latest_forward_obstacle.blocked=true` 时强制关闭巡线控制、清零命令并发布停止。
- 用户点击 STOP、手动方向键、发送导航目标、巡逻启动时会覆盖巡线控制。

## 数据流

1. `camera_loop()` 更新 `latest_camera_jpeg`。
2. `/api/line_follow/status` 解码最新 JPEG，调用 `LineFollower.process()`。
3. 返回 JSON：参数、结果、控制状态和安全状态。
4. `/api/line_follow/debug.jpg` 返回叠加后的 JPEG。
5. `/api/line_follow/config` 保存参数；如果启用控制，后台 `line_follow_loop()` 周期性根据最新帧更新 `/cmd_vel` 状态。

## 测试

核心算法用 `unittest` 构造合成图像：

- 黑线在画面中心时，检测成功且偏差接近 0。
- 黑线在左侧时，偏差为负。
- 黄线可被 HSV 阈值识别。
- 空白图像返回丢线状态和停止命令。
- 左右候选同时出现时，上一帧稳定方向不会被单帧噪声反向覆盖。
- 分叉锁定期间黄/红禁压接触不会反向覆盖锁定角速度，临时相反路径也不会污染上一分支方向。
- 黄/红边界明显弯曲时，居中的近场道路会得到小幅顺弯预测。
- 禁压接触会覆盖方向预测并停车。
- 参数校验会限制线速度和角速度范围。

集成验证：

- `python3 -m unittest tests.test_line_follow -v`
- `python3 -m unittest tests.test_patrol -v`
- `python3 -m py_compile web_panel/line_follow.py web_panel/server.py`
- 小车端重启 `ucar_web.service` 后访问 `http://10.68.225.179:8080`，先看巡线预览，不立即启用控制。
